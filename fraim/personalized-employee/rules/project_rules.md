# Project Rules — IntegrateAPIweb

## Product & Design

1. **Marketing website rebuild.** Prioritize visual quality and messaging clarity above all else.
2. **Darker grey-based palette.** Shift from the current white/light-blue to a darker, grey-based theme while keeping the clean, minimal feel.
3. **Sales tool for IntegrateAPI.** The site exists to convert visitors into leads for IntegrateAPI's custom automation services.
4. **Non-technical audience.** Content should speak to small business owners who are not technical. No jargon, no complexity.
5. **Mobile-first responsive design.** Every page must work well on phones, tablets, and desktops.
6. **Copy requires user approval.** All messaging and copy must be reviewed by the user before finalizing. Messaging is a key change area in this rebuild.
7. **No backend unless required.** No database or server-side logic unless contact forms specifically require it.

## Workflow & Process

8. **All UI changes must use a UI/UX skill.** Use `/ui-ux-pro-max` or its sub-skills for any visual or interface work. Research what "vibe coded" websites look like and actively avoid that aesthetic.
9. **All code changes must trigger a codex review loop.** Use `/codex-review` after every implementation to verify quality before marking work complete.
10. **Final site tested end-to-end with Chrome DevTools MCP.** Simulate real user interactions across all pages and functionality using the Chrome DevTools MCP tools.
11. **Always trigger `/discussion` before making changes.** Talk through proposed changes with the user first. No implementation without alignment. The full workflow is: `/discussion` -> implement (with UI/UX skills) -> `/codex-review` -> test with Chrome DevTools MCP.
